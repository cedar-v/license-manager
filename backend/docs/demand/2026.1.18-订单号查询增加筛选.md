# 订单号查询增加筛选

改造接口：`GET /api/cu/orders`

## 背景
当前订单列表仅支持分页查询，需要增加筛选能力以提升定位订单效率。

## 目标
在现有订单列表接口基础上，增加 3 个可选筛选字段：`search`、`status`、`time`，并保持向后兼容（不传筛选参数时行为不变）。

## 接口定义
### 请求
`GET /api/cu/orders`

#### Query 参数
- `page`：页码，默认 `1`
- `page_size`：每页数量，默认 `10`，最大 `100`
- `search`：模糊匹配关键字（订单号 `order_no` 或订单授权码 `authorization_code`）
- `status`：订单状态筛选（等值匹配）
  - 目前系统内常见值：`pending` / `paid` / `cancelled`
- `time`：创建时间筛选（按订单 `created_at`）
  - 支持值（推荐）：`today` / `week` / `month` / `three_months`
  - 支持中文别名：`今天` / `本周` / `本月` / `近三个月`

#### 过滤规则
- `search`：`order_no LIKE %search% OR authorization_code LIKE %search%`
- `status`：`status = ?`
- `time`：对 `created_at` 增加范围过滤（左闭右开：`>= start AND < end`）
  - `today/今天`：当天 00:00:00 ~ 次日 00:00:00
  - `week/本周`：以周一为一周起始
  - `month/本月`：当月 1 日 00:00:00 ~ 下月 1 日 00:00:00
  - `three_months/近三个月`：从当前时间往前推 3 个月（按日期对齐到 00:00:00）~ 次日 00:00:00

### 响应
保持现有返回结构不变：`models.CuOrderListResponse`

## 示例
- 订单号/授权码模糊搜索：
  - `GET /api/cu/orders?page=1&page_size=10&search=ORD20260118`
- 按状态筛选：
  - `GET /api/cu/orders?status=pending`
- 按时间筛选（近三个月）：
  - `GET /api/cu/orders?time=three_months`
- 组合筛选：
  - `GET /api/cu/orders?search=ORD&status=paid&time=month`
