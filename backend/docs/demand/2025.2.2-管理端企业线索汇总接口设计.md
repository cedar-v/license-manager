# 管理端企业线索汇总接口设计

## 接口路径
`GET /api/leads/summary`

## 请求
不需要任何参数。

## 鉴权
需要管理端权限（与线索列表/详情接口一致）。

## 响应示例
```json
{
  "code": "000000",
  "message": "success",
  "data": {
    "total_count": 1200,
    "pending_count": 300,
    "following_count": 520,
    "converted_count": 260,
    "lost_count": 120
  },
  "timestamp": 1738454400
}
```

## 字段说明
- total_count：线索总数
- pending_count：待跟进数量
- following_count：已跟进数量
- converted_count：已成交数量
- lost_count：已流失数量
