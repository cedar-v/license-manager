# 令牌刷新

目前系统有一个超管端和一个用户端，需要对两个端进行令牌刷新机制的改造。

Token过期与刷新机制
Token刷新是为了在保证安全性的同时提升用户体验，避免频繁登录。核心是使用短期访问令牌(Access Token) + 长期刷新令牌(Refresh Token) 的双令牌策略。
基本流程
1. 用户登录
用户提交账号密码 → 后端验证 → 返回两个token：
- Access Token (短期，如15分钟-2小时)
- Refresh Token (长期，如7-30天)
2. 日常请求
前端携带Access Token → 后端验证 → 返回数据
3. Token过期处理
Access Token过期 → 请求失败(401) → 前端自动用Refresh Token换新的Access Token → 重试原请求
